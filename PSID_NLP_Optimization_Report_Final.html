Method Summary and Formula Definitions

Questions are ranked with a utility-to-burden ratio that balances analytical value against respondent effort. Utility is driven by taxonomy-weighted crisis keywords. Burden penalizes longer wording and question complexity so that high-value, shorter questions can rise to the top.

Ranking score

Ri = Ui / Bi

Ri is the final ranking score for question i. Higher values indicate better information return per unit of burden.

Utility definition

Ui = Σ wk

Ui sums taxonomy impact weights for the crisis-relevant keywords extracted from the question text. More consequential constructs receive larger weights.

Burden definition

Bi = max(αNi + βCi, 0.1)

Bi scales with question length Ni and structural complexity Ci, with a minimum floor so the ratio remains stable.

Interpretation and Recommendations

Interpretation

Three conclusions remain stable after the final refresh. First, the Generic Core is intentionally small and should stay that way. Second, Katrina materially changes the value of the Pandemic / Disaster toggle by adding high-value physical-shock content. Third, the 29.17-minute final benchmark should be treated as the upper deployment bound rather than something to expand casually.

Recommendations

  • Keep the ranked 28-question analytic module as the benchmark production artifact.
  • Use the HTML questionnaire demos and dashboard for stakeholder communication, not the raw markdown export.
  • Regenerate future figures and report tables only through the maintained notebook refresh path and artifact generator.
  • If shortening is required, review the longest Katrina-derived items first instead of trimming the Generic Core.

Workflow alignment

This web report matches the current artifact workflow: `generate_psid_artifacts.py` refreshes the figures, summary JSON, and dashboard bundle; the final notebook triggers the same path; and the dashboard, master questionnaire viewer, and report all now point to the same final public deliverables.
